# codex-dock 新手使用指南

## 1. 这是什么

`codex-dock` 是一个本地多账号切换工具，用来管理 Codex / ChatGPT 桌面端的登录状态。它的核心能力有：

- 保存当前登录账号，按别名归档
- 在多个已保存账号之间切换
- 回到默认干净状态，方便重新登录新账号
- 查看额度、套餐、重置时间
- 通过网页面板完成大部分操作

当前项目已安装到：

`E:/AIProject/CodexAccountSwitch`

---

## 2. 使用前提

开始前先确认这几件事：

1. 电脑已经安装 `Python 3.10+`
2. 你本机已经安装并能打开 Codex 桌面端
3. 你至少已经登录过一个账号，且本机存在 Codex 登录信息
4. 不要在 Codex 软件里直接点“退出登录”，否则切换流程会变蠢

这个工具默认会读取：

- 当前登录：`C:/Users/Administrator/.codex/auth.json`
- 已保存账号归档：`C:/Users/Administrator/.codex/codex-dock/`
- 本工具本地记录：`E:/AIProject/CodexAccountSwitch/config/accounts.json`

如果你手动设置了 `CODEX_HOME`，这些路径会跟着变。

## 4. 新手第一次怎么用

这是最关键的流程，按顺序来，别乱点。

### 场景：先保存当前账号，再去登录第二个账号

1. 先确保 Codex 桌面端当前已经登录了一个账号
2. 启动 `codex-dock`
3. 在网页面板点击“添加当前账号”，或者在 CLI 里选 `2`
4. 给这个账号起一个好认的别名，比如：
   - `work`
   - `personal`
   - `team`
5. 保存成功后，再切换到“默认 / 干净状态”
6. 重新打开 Codex，登录另一个账号
7. 再次运行 `codex-dock`，把第二个账号也保存进去

做完之后，你就能在这些账号之间来回切换。

## 5. 网页面板能干什么

网页面板是主力入口，建议优先用它。

你可以在里面完成这些操作：

- 查看当前账号邮箱、套餐、额度、重置时间
- 查看所有已保存账号的卡片列表
- 添加当前账号
- 删除某个已保存账号
- 切换到某个已保存账号
- 切换回默认干净状态
- 刷新额度
- 精准刷新额度
- 隐藏或显示邮箱

如果当前登录账号还没保存，面板也会提示你。

---

## 6. CLI 菜单怎么理解

CLI 适合快速操作，菜单含义如下：

- `1`：刷新本地额度快照，快，但可能不够准
- `2`：保存当前登录账号
- `3`：删除已保存账号
- `4`：切换账号，或者切回默认环境
- `5`：重置为干净会话
- `p`：精准刷新当前账号额度
- `w`：打开网页面板
- `q`：退出

建议：

- 日常管理用网页面板
- 临时保存、快速切换用 CLI

## 7. 这个工具会往哪里写数据

你需要知道它改了哪些地方：

### 本项目目录内

- `E:/AIProject/CodexAccountSwitch/config/accounts.json`
  - 保存账号别名、邮箱、额度快照等映射信息

### Codex 本地目录内

- `C:/Users/Administrator/.codex/auth.json`
  - 当前正在使用的登录态
- `C:/Users/Administrator/.codex/codex-dock/<别名>/auth.json`
  - 已保存账号的认证归档

这意味着：

- 切换账号，本质上就是替换当前的 `auth.json`
- 账号归档默认都只保留在本机

## 9. 常见问题

### 9.1 提示找不到 Python

说明 `python` 或 `py` 没进系统 `PATH`。先安装 Python，再重新打开终端。

### 9.2 切换账号后 Codex 没反应

可能是桌面端缓存把登录态顶回去了。处理顺序：

1. 先切回默认干净状态
2. 关闭 Codex
3. 重新打开 Codex
4. 再执行一次切换

### 9.3 CLI 里显示的额度和网页不一致

正常。CLI 里快速刷新读的是本地快照，网页里的精准刷新会更准。

### 9.4 为什么不建议直接点 Logout

因为这个工具的设计逻辑是“保留当前登录态 -> 归档 -> 切换”。你直接 Logout，登录态没了，后面保存和切换都会变得很别扭。

## 10. 建议的使用习惯

给你一套最省事的习惯：

1. 新账号第一次登录后，立刻保存成别名
2. 别名统一用简单英文，例如 `work`、`home`、`team`
3. 日常切换优先走网页面板
4. 只在你确定需要时，才装全局命令
5. 定期看一下 `config/accounts.json` 和 `.codex/codex-dock/` 是否正常存在
